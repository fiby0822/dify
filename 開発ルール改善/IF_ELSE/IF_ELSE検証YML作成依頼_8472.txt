# Dify YMLファイル作成依頼プロンプト

## 要件
IF_ELSEノードのコーディングルール検証用のワークフローを作成してください。

### ワークフロー概要
「商品在庫管理と注文処理判定システム」を作成します。
このシステムは、商品の在庫状況と注文金額に基づいて、適切な処理ルートを判定します。

### 処理フロー
1. ユーザーから商品名、注文数量、単価を入力
2. 在庫数をチェック（仮想的に100個と設定）
3. 注文金額を計算（数量×単価）
4. IF_ELSEノードで以下の条件分岐を実行：
   - ケース1：在庫が十分（注文数量 <= 100）かつ高額注文（金額 >= 10000）
   - ケース2：在庫が十分（注文数量 <= 100）かつ通常注文（金額 < 10000）
   - デフォルト：在庫不足（注文数量 > 100）
5. 各ケースに応じたメッセージを返す

### IF_ELSEノード検証項目
1. 条件IDの必須設定（各条件に一意なIDを付与）
2. case_idフィールドの必須設定（idと同じ値）
3. ASCII文字の比較演算子使用（"<="、">="、"<"）
4. 複数条件のAND演算
5. 数値比較の実装
6. エッジのsourceHandleとcase_idの正しい接続

### 技術要件
- app.mode: advanced-chat
- version: 0.3.0
- 全てのノードに適切なposition、height、widthを設定
- graphセクションを使用
- YAMLコメントは一切含めない
