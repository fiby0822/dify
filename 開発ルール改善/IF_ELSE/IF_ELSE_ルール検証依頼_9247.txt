# Dify YMLファイル作成依頼プロンプト

## 要件
IF_ELSEコンポーネントの新しいルールに基づいた動作検証用のワークフローを作成してください。

### ワークフローの概要
「注文処理自動分類システム」を作成します。このシステムは、ユーザーから入力された注文情報（金額、会員種別、配送方法）を基に、適切な処理ルートに振り分けます。

### 主要機能
1. **注文金額による処理分岐**
   - 10,000円以上：優先処理
   - 5,000円以上10,000円未満：通常処理
   - 5,000円未満：簡易処理

2. **会員種別による追加判定**
   - プレミアム会員：特別割引適用
   - 通常会員：標準処理
   - 非会員：会員登録案内

3. **配送方法チェック**
   - express：即日配送手配
   - standard：通常配送
   - NULL/未指定：デフォルト配送

### 技術的検証要件
1. **Unicode演算子の使用**
   - 数値比較に≥、≤を使用
   - 一部でASCII代替（gte、lte）も検証

2. **case_idの正しい設定**
   - すべてのケースでidとcase_idを一致させる

3. **複数条件の論理演算**
   - AND条件：金額と会員種別の組み合わせ
   - OR条件：特別処理の条件

4. **変数参照エラーの防止**
   - 各分岐で独立したAnswerノードを使用
   - 実行されない分岐の変数を参照しない

5. **NULL/存在チェック**
   - 配送方法のNULLチェック実装

### ワークフローの流れ
1. Start：ユーザー入力の受付
2. Code：入力データの解析と整形
3. IF_ELSE（金額判定）：注文金額による第一次分岐
4. IF_ELSE（会員判定）：会員種別による第二次分岐
5. IF_ELSE（配送判定）：配送方法による最終分岐
6. Answer：各ルートに応じた結果表示

### 注意事項
- advanced-chatモードを使用
- YAMLコメントは一切含めない
- エッジのisInLoopは必ずfalseに設定
- app.use_icon_as_answer_iconを必ず設定
- ファイルサイズは500行以内に収める
