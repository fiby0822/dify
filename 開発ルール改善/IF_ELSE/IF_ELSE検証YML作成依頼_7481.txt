# Dify YMLファイル作成依頼プロンプト

## 要件
更新されたIF_ELSEコンポーネントの動作検証用YMLファイルを作成してください。
このYMLファイルは、コンポーネント記述ルール_IF_ELSE.txtの最新版に記載されているルールが正しく動作することを確認するためのテストケースです。

### 検証項目：
1. **YAMLコメントの除外**
   - YAMLファイル内にコメント（#で始まる行）を一切含めない
   - 説明はdescフィールドで代替する

2. **エッジ接続の正確性**
   - targetTypeに実際のノードタイプを使用（"custom"ではなく"answer"や"llm"など）
   - sourceHandleにケースのidを正しく使用

3. **フィールド名の正確性**
   - isInLoopフィールドを使用（isInIterationは使用しない）
   - 必須フィールドの完全性（height, width, positionAbsolute等）

4. **新しい比較演算子の検証**
   - "in", "not in": リスト内存在チェック
   - "all of": すべて含むチェック
   - "exists", "not exists": 存在チェック

5. **オプションフィールドの使用**
   - conditions内のidフィールド（オプション）
   - varTypeフィールド（例: "file"）

6. **advanced-chatモードでの実装**
   - graphセクションを使用した実装
   - ノードとエッジの完全な定義

### テストシナリオ：
「ファイル分析システム」を作成してください。
- ファイルタイプの判定（テキスト、画像、動画、その他）
- ファイルサイズによる処理分岐（1MB以下、1-10MB、10MB以上）
- 複数条件での分析（ファイルタイプとサイズの組み合わせ）
- 存在チェックとエラーハンドリング

各分岐は独立したanswerノードに接続し、適切なメッセージを返すようにしてください。

### 特に注意すべき点：
1. YAMLコメント（#で始まる行）は絶対に含めないこと
2. IF_ELSEノードでは必ずidとcase_idの両方を設定すること
3. 数値比較にはUnicode演算子（≥、≤、≠）を使用すること
4. targetTypeには接続先ノードの実際のdata.typeを使用すること
5. エッジのsourceHandleには必ずケースのidを使用すること
6. advanced-chatモードではgraphセクションを必ず使用すること
