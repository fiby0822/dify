# Dify YMLファイル作成依頼プロンプト

## 要件
更新されたIF/ELSEコンポーネントの記述ルールが正しく機能するかを検証するため、以下の要素を徹底的にテストするDifyアプリケーションを作成してください：

1. **必須フィールドの検証**
   - idとcase_idの両方を必ず設定（同じ値）
   - 外側のtype属性は"custom"、data.typeは"if-else"
   - data.selectedとdata.titleの必須設定

2. **Unicode文字の比較演算子テスト**
   - "≥"（以上）を使用した数値比較
   - "≤"（以下）を使用した数値比較
   - "≠"（等しくない）を使用した比較
   - "="（等しい）を使用した比較

3. **文字列比較演算子テスト**
   - contains（部分一致）
   - not contains（部分不一致）
   - start with（前方一致）
   - end with（後方一致）
   - is/is not（完全一致/不一致）

4. **論理演算子の組み合わせ**
   - AND条件での複数条件判定
   - OR条件での複数条件判定

5. **エッジケースとデフォルト処理**
   - 条件に一致しない場合のデフォルトケース
   - sourceHandleとcase_idの正確な対応

## 重要な注意事項（必須）
- **case_idフィールドは必須**：各ケースにidとcase_idの両方を設定し、同じ値にすること
- **Unicode文字の正確な使用**：
  - ASCII文字の">="ではなく、Unicode文字の"≥"を使用
  - ASCII文字の"<="ではなく、Unicode文字の"≤"を使用
  - ASCII文字の"!="ではなく、Unicode文字の"≠"を使用
- **外側のtype属性**：必ず"custom"を使用（"if-else"ではない）
- **data.type属性**：必ず"if-else"を使用
- **必須フィールド**：height、width、positionAbsolute、data.selected、data.titleを必ず設定
- **エッジのsourceHandle**：ケースのidと完全に一致させること
