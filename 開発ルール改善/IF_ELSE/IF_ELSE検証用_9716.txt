# Dify YMLファイル作成依頼プロンプト

## 要件
IF_ELSEコンポーネントの記述ルールに従って作成されたYMLファイルが正しく動作するかを検証するためのテストアプリケーションを作成してください。

以下の検証項目をすべて含むワークフローを作成してください：

1. **文字列比較の検証**
   - "contains"（部分一致）のテスト
   - "is"（完全一致）のテスト
   - "start with"（前方一致）のテスト
   - "not contains"（部分不一致）のテスト

2. **数値比較の検証**（Unicode演算子を正しく使用）
   - "≥"（以上）のテスト
   - "≤"（以下）のテスト
   - "≠"（等しくない）のテスト
   - ">"（より大きい）のテスト

3. **NULL/存在チェックの検証**
   - "empty"（空）のテスト
   - "not empty"（空でない）のテスト
   - "exists"（存在する）のテスト

4. **複数条件の検証**
   - AND条件（logical_operator: "and"）のテスト
   - OR条件（logical_operator: "or"）のテスト

5. **必須パラメータの検証**
   - case_idが正しく設定されているか
   - heightとwidthが適切に設定されているか
   - positionAbsoluteが正しく設定されているか
   - data.selectedとdata.titleが設定されているか

各テストケースの結果を確認できるよう、適切な出力ノードも配置してください。

## コーディングルール
Dify YMLファイルの作成にあたっては、以下のコーディングルールに従ってください：

### 1. 全体コーディングルール
/Users/tatsuya/Documents/01_fiby/09_ゼット/コーディングルール/全体コーディングルール/全体コーディングルール.md を参照してください。

### 2. コンポーネント別コーディングルール
各コンポーネントを使用する際は、/Users/tatsuya/Documents/01_fiby/09_ゼット/コーディングルール/コンポーネント別コーディングルール フォルダ内の該当するtxtファイルを参照してください。

特に、IF_ELSEコンポーネントについては以下の点に注意してください：
- 外側のtype属性は必ず"custom"
- data.typeが"if-else"
- 各ケースにidとcase_idの両方が必須（同じ値を設定）
- 数値比較ではUnicode文字を使用（"≥"、"≤"、"≠"）

### 3. 品質チェック
YMLファイル作成後は、/Users/tatsuya/Documents/01_fiby/09_ゼット/コーディングルール/品質チェック/品質チェックリスト.md を参照し、必ずセルフレビューを実施してください。
