# Dify YMLファイル作成依頼プロンプト

## 要件
IF_ELSEコンポーネントの動作検証用ワークフローを開発してください。
年収に基づいて税率区分を判定し、適切な節税アドバイスを提供するシステムです。

### 機能要件
1. **入力受付機能**
   - ユーザーから年収情報を受け取る
   - 数値形式での入力を想定

2. **年収解析機能**
   - 入力された年収を解析
   - 数値を文字列として抽出・整形

3. **税率区分判定機能（IF_ELSE使用）**
   - 低所得層：195万円以下
   - 中所得層：195万円超〜330万円以下  
   - 高所得層：330万円超
   - 各区分に応じた処理への分岐

4. **個別アドバイス生成機能**
   - 低所得層向け：基礎控除や各種控除の活用アドバイス
   - 中所得層向け：ふるさと納税やiDeCo等の活用アドバイス
   - 高所得層向け：法人設立や不動産投資等の高度な節税アドバイス

5. **結果出力機能**
   - 各分岐で生成されたアドバイスを適切に表示
   - 独立実行パターンで各分岐が完結

### 技術的制約
- **Codeノードは使用しない**こと
- IF_ELSEノードの検証が主目的のため、条件分岐を必ず含める
- 比較演算子はASCII文字（>=、<=、!=）のみ使用
- 独立実行パターンを採用（各分岐が独立して完結）

### IF_ELSE検証ポイント
1. case_idとsourceHandleの完全一致
2. targetTypeの正確な指定
3. デフォルトケース（false）の適切な処理
4. 各条件が重複しないこと
5. 条件分岐後の変数参照が正しいこと
