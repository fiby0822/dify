# Dify YMLファイル作成依頼プロンプト

## 要件
IF/ELSEコンポーネントの動作検証用YMLファイルを作成してください。
以下の機能を実装し、コンポーネント記述ルールが正しく動作することを確認します：

1. **基本的な条件分岐テスト**
   - ユーザー入力（数値）を受け取る
   - 数値の大小による3段階の分岐（0-30：低、31-70：中、71-100：高）
   - 各分岐で異なるメッセージを生成

2. **文字列条件のテスト**
   - ユーザー入力（文字列）を受け取る
   - "緊急"を含むかどうかの判定
   - 完全一致、部分一致の両方をテスト

3. **複数条件の組み合わせテスト**
   - 数値と文字列の両方を評価
   - AND条件での判定

4. **エラー防止パターンの検証**
   - 独立answer型パターンを使用
   - 各分岐から直接answerノードへ接続
   - 未実行ノードの参照エラーを回避

5. **必須フィールドの確認**
   - case_idとidの両方を正しく設定
   - Unicode演算子（≥、≤、≠）の使用
   - 全ての必須パラメータを含める

### 4. 特に注意すべき点

- case_idフィールドを必ず設定（idと同じ値）
- 数値比較にはUnicode文字を使用（≥、≤、≠）
- 外側のtypeは"custom"、data.typeが"if-else"
- height、width、positionAbsolute、selected、titleは必須
- エッジのtargetTypeには実際のノードタイプを指定
