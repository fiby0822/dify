# Dify YMLファイル作成依頼プロンプト

## 要件
IF_ELSEコンポーネントの記述ルールが正しく動作することを検証するためのDify YMLファイルを作成してください。

以下の要素を必ず含めて、ルールの全ての側面を検証できるようにしてください：

1. **必須パラメータの検証**
   - 外側のtypeが"custom"、data.typeが"if-else"であること
   - 各ケースにidとcase_idの両方が設定されていること（同じ値）
   - height, width, positionAbsolute等の必須フィールドが設定されていること
   - data.selectedとdata.titleが含まれていること

2. **比較演算子の動作検証**
   - Unicode文字の数値比較演算子（≥、≤、≠）を使用したケース
   - 文字列比較演算子（contains, is, start with等）を使用したケース
   - NULL/存在チェック演算子（empty, not empty, exists等）を使用したケース

3. **複数条件と論理演算子の検証**
   - 複数条件をAND条件で結合したケース
   - 複数条件をOR条件で結合したケース

4. **エッジ接続による分岐動作の検証**
   - 各ケースが正しく次のノードに接続されること
   - sourceHandleにケースのidが正しく設定されること

5. **エラーになりやすい部分の確認**
   - 比較演算子でASCII文字（">="等）を使わず、Unicode文字（"≥"等）を使用
   - case_idフィールドが欠落していないこと

実際の業務で使用することを想定し、以下のシナリオでワークフローを構築してください：
- ユーザーからの入力値（数値）を受け取る
- その値に基づいて3つ以上の分岐を作成（高・中・低など）
- 各分岐で異なる処理を実行する
