# Dify YMLファイル作成依頼プロンプト

## 要件
IF/ELSEコンポーネントの記述ルールが正しく動作するかを検証するためのDifyアプリケーションを作成してください。
以下のすべての機能と条件分岐パターンを網羅的にテストする必要があります：

### 検証項目
1. **基本的な条件分岐**
   - 単一条件での分岐（is演算子）
   - 数値比較での分岐（>, <, >=, <=）
   - 文字列の部分一致での分岐（contains, not contains）

2. **複数条件の組み合わせ**
   - AND条件での分岐（logical_operator: and）
   - OR条件での分岐（logical_operator: or）
   - 3つ以上の分岐ケース

3. **必須パラメータの検証**
   - すべての必須パラメータが正しく設定されているか
   - type属性が"if-else"になっているか（外側のtypeは"custom"）
   - 各ケースにユニークなidが設定されているか

4. **エッジ接続の検証**
   - 各分岐先が正しく接続されているか
   - sourceHandleで分岐先が明確に指定されているか
   - デフォルトケースの処理

### アプリケーションの内容
以下の流れで動作する質問応答分類アプリケーションを作成してください：
1. ユーザーからの質問を受け取る
2. 質問内容を分析するLLMノード（カテゴリとスコアを出力）
3. 分析結果に基づいてIF/ELSEで分岐：
   - ケース1: 技術的質問（カテゴリが"技術"を含み、かつスコア>70）
   - ケース2: 一般的質問（カテゴリが"一般"を含む、またはスコア<=70）
   - ケース3: 緊急対応（カテゴリが"緊急"を含む）
   - デフォルト: その他の質問
4. 各分岐先で異なる回答を生成
5. 最終的な回答を表示

各条件分岐で異なる比較演算子と論理演算子を使用し、IF/ELSEコンポーネントのすべての機能が正しく動作することを確認できるようにしてください。
