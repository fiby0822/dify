# Dify YMLファイル作成依頼プロンプト

## 要件
LLMコンポーネントの記述ルールが正しく動作するかを検証するため、以下の要素を含むDifyアプリケーションのYMLファイルを作成してください：

### 検証対象項目
1. **必須パラメータの完全性**
   - 全ての必須パラメータが正しく設定されていること
   - 外側のtype属性が"custom"固定であること
   - data内のtype属性が"llm"であること
   - positionAbsoluteがpositionと同じ値であること
   - height/widthが適切に設定されていること

2. **プロンプトテンプレートの正確性**
   - id属性がユニークで設定されていること
   - systemロールとuserロールの両方を使用すること
   - 変数参照（{{#node_id.variable_name#}}形式）を含むこと

3. **必須フィールドの設定**
   - structured_output_enabled: false
   - vision.enabled: false
   - contextフィールドの設定

4. **オプションパラメータの動作**
   - completion_params（temperature、max_tokens等）
   - memory設定

### アプリケーション概要
カスタマーサポート向けの問い合わせ対応アプリケーションを作成します。
- 開始ノードでユーザーの問い合わせを受け取る
- LLMノードで問い合わせに対する回答を生成する
- 終了ノードで回答を出力する
