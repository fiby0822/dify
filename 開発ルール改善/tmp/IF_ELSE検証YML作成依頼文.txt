# Dify YMLファイル作成依頼プロンプト

## 要件
顧客満足度評価システムを作成してください。
このシステムは、顧客満足度スコア（0-100点）を入力として受け取り、IF_ELSEノードで3段階の条件分岐を行い、それぞれの評価レベルに応じた適切なフィードバックとアクションプランを提供します。

### 詳細仕様
1. **入力**：顧客満足度スコア（0-100の数値）
2. **条件分岐（IF_ELSEノード）**：
   - 80点以上：「優良顧客」カテゴリー → 感謝メッセージと特典案内
   - 50-79点：「通常顧客」カテゴリー → 標準的なフォローアップメッセージ
   - 50点未満：「改善必要」カテゴリー → 改善提案とサポート案内
3. **出力**：各カテゴリーに応じたメッセージとアクションプラン
4. **必須条件**：Codeノードは使用しないこと

### 4. 重要な注意事項
- version: 0.3.0 を必ず使用すること
- IF_ELSEノードの実装では以下を厳守：
  - 各ケースにlogical_operatorを必ず設定（"and"または"or"）
  - case_idとidの両方を設定し、同じ値にする
  - 比較演算子はASCII文字のみ使用（">="、"<="、"!="）
  - Unicode文字（≥、≤、≠）は絶対に使用しない
  - エッジのsourceHandleはcase_idまたは"false"を正確に設定
- YAMLファイル内にコメント（#で始まる行）を一切含めないこと
- advanced-chatモードを使用し、startノードのvariablesは空配列[]にすること
